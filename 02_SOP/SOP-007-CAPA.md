## 1. Purpose

建立纠正措施和预防措施流程，确保系统性、重复性或高风险问题得到根因分析、措施实施、有效性验证和关闭。

## 2. CAPA Triggers

CAPA 可由以下来源触发：

- 重复或严重不合格
- 客户投诉或不良事件趋势
- 内审或外审发现
- 供应商质量问题
- 验证确认失败
- 风险控制措施失效
- 管理评审决议
- 法规、标准或客户要求变化

## 3. Procedure

### 3.1 Initiation

质量负责人评估问题严重性、频次、范围和风险，决定是否启动 CAPA。

### 3.2 Containment

必要时立即采取遏制措施，例如暂停放行、隔离批次、通知客户、增加检验或暂停供应商。

### 3.3 Root Cause Analysis

根因分析应使用合适方法，例如 5 Why、鱼骨图、过程流程分析、批记录回溯、供应商调查或实验验证。

### 3.4 Action Plan

行动计划应明确：

- 纠正措施
- 预防措施
- 责任人
- 完成期限
- 需更新的文件、培训、验证或风险文件

### 3.5 Effectiveness Check

CAPA 关闭前应验证措施有效性。有效性证据可以包括复验合格、趋势改善、审核通过、过程能力提升或投诉下降。

## 4. Linkage

CAPA 应按需要链接到：

- 风险管理文件
- 设计变更
- 工艺变更
- 供应商纠正措施
- 管理评审
- 培训记录

## 5. Records

- CAPA Request
- Root Cause Analysis
- CAPA Action Plan
- Effectiveness Evidence
- CAPA Closure Approval
