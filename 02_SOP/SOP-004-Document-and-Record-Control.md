## 1. Purpose

确保质量体系文件和研发记录被批准、发布、修订、归档和检索。

## 2. Document States

- Draft: 草稿，不能作为正式执行依据
- In Review: 审核中
- Approved: 已批准，可执行
- Obsolete: 作废，仅保留历史记录

## 3. Minimum Metadata

每个受控文件应包含：

- 文件编号
- 标题
- 版本
- 状态
- 所有人
- 批准人
- 生效日期
- 变更历史

## 4. GitHub Workflow

建议流程：

1. 在功能分支修改文件。
2. Pull Request 中说明修改原因和影响范围。
3. 质量负责人、流程负责人或指定 CODEOWNER 审核。
4. 合并到主分支后视为发布，除非企业另有电子签名系统。
5. 重大文件变更同步更新培训要求。

## 5. Record Retention

记录保存期限应满足目标市场法规、客户协议和企业程序要求。与注册、灭菌、投诉、CAPA、批次追溯相关记录应特别控制。
