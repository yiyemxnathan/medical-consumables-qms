## 1. Purpose

定义一次性医疗器械研发验证与确认活动的计划、执行、偏差处理和报告要求。

## 2. Verification vs Validation

- Verification: 证明设计输出满足设计输入，例如材料克重、阻隔性能、尺寸、公差、封口强度。
- Validation: 证明产品满足预期用途和用户需求，例如临床使用模拟、穿脱体验、铺单展开、无菌开启、运输后完整性。

## 3. Test Planning

验证确认计划应明确：

- 测试目的
- 对应设计输入或风险控制措施
- 样品来源、批次和数量
- 测试方法和适用标准
- 接收准则
- 实验室或责任人
- 偏差处理方式
- 报告输出

## 4. Typical Test Matrix

| Product | Typical Tests |
| --- | --- |
| Surgical gown | 抗渗水、抗合成血液、拉伸、胀破、接缝强度、微生物屏障、洁净度、掉絮、舒适性、穿脱确认 |
| Surgical drape | 阻液、吸液/反渗、拉伸、贴膜粘附、洞口定位、铺设确认、洁净度 |
| Isolation gown | 液体屏障、拉伸、接缝、袖口、穿脱污染风险、尺寸覆盖性 |
| Medical mask | BFE/PFE、压差、合成血液穿透、阻燃、微生物清洁度、耳带/绑带强度 |
| Gloves | AQL 破洞、拉伸、老化后拉伸、粉末/无粉、蛋白/化学残留、生物相容性、穿戴确认 |
| Sterilization wrap | 微生物屏障、透气性、胀破/拉伸、抗撕裂、包裹完整性、灭菌适用性、老化和运输模拟 |

## 5. Deviations and Failures

测试失败或偏差应记录并评估：

- 是否为样品、方法、设备、人员或设计本身问题
- 是否影响已完成测试结果
- 是否需要重新测试或扩大样本量
- 是否触发风险文件更新、设计变更或 CAPA
